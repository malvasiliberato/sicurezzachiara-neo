# Security Policy

## Supported Versions
Il progetto e' in sviluppo attivo sul branch principale corrente.

## Reporting a Vulnerability
Non aprire issue pubbliche con dettagli sensibili.

Inviare una segnalazione privata a:
- `liberato.malvasi@gmail.com`

Includere almeno:
- descrizione del problema
- impatto atteso
- passi per riprodurlo
- eventuale proof of concept minima

## Scope
Sono considerate rilevanti in particolare:
- esposizione di segreti o credenziali
- bypass di autenticazione o autorizzazione
- problemi di isolamento tenant
- vulnerabilita' nelle dipendenze Composer/NPM
